Top 3 Process Mining Use Cases

1. Ethoca Chargeback & Dispute Resolution — Prove OEM Value Before the Contract Closes

The Ethoca OEM Native ISV opp is in Assess, which means the SF/Mastercard team is still building the "why now" narrative for embedding Ethoca into Financial Services Cloud. AFPI mines the actual dispute lifecycle event log — issuer intake, Ethoca alert, merchant refund path, network representment — to expose which variants leak days and dollars, then hands the top three broken paths to Agentforce Operations. That turns the OEM story from "another data feed" into a two-to-four-week measurable cycle-time and write-off reduction win. Metric: dispute-to-resolution days, first-cycle refund rate, and chargeback write-off dollars.

Key System(s) to be Process Mined: Ethoca, Financial Services Cloud

2. Merchant & Acceptance Onboarding — Network-Scale Rework Reduction

Mastercard's Commercial Acceptance Solutions team owns a very high-volume merchant/acquirer onboarding book with global compliance overlay (PCI DSS, sanctions, market-specific KYC). 3. FSC Data Cloud — Process-Lineage Layer for Issuer Analytics

The Mastercard FSC Data Cloud ISV opp is at the earliest stage; the fastest way to accelerate it is to give the pilot a use case only Data Cloud + AFPI can solve. AFPI overlays process lineage on the issuer-servicing event stream (auth → decline → chargeback → retention save) so Mastercard's issuer customers can see not just the data, but the actual sequence of steps that produced the outcome — a dimension Celonis-style tools miss because they treat network telemetry as SQL, not process. Differentiates AFPI as the collapse of the SQL+PQL two-specialist model into one Ops analyst. Metric: auth-approval rate improvement per issuer, false-decline rate, retention-save conversion.

Top 3 Compliance Center Use Cases

1. DORA ICT Resilience & Third-Party Register (EU)

Mastercard operates under DORA in the EU: ICT resilience testing, third-party register, incident classification, and reporting all require defensible "prove the process" evidence. Compliance Center codifies the resilience-testing and incident-response workflows as a controlled process with sign-offs, evidence capture, and codified SLAs — audit-defensible without a PQL specialist writing custom queries. Lands as a natural companion to the FSC Data Cloud pilot: same data, one more use case that pays back inside the quarter. Metric: reporting-cycle days, incident-classification accuracy, examiner-finding count.
