Top 5 Process Intelligence Use Cases to be Mined

1. Bit2win CPQ Replacement — Baseline the Quote-to-Contract Process Before You Rebuild

Axpo Italia is actively replacing Bit2win CPQ ($270k opp in stage 02) — the quote-to-contract flow for Flexy Mix, Smart Portfolio, and Corporate PPA products spans MuleSoft, Bit2win, and Sales/Service Cloud, with heavy pricing-desk touchpoints for structured energy products. Rebuilding on a new CPQ without first mining the as-is flow risks re-implementing the same rework loops (pricing exceptions, contract redlines, credit checks). AFPI mines the current event log to expose which product variants generate the most cycle-time drag and hand-offs, so the new CPQ is designed against real friction — not the sales team's memory. Metric moved: quote-to-signed cycle time, pricing-exception rate, deal-desk touches per quote.

Key System(s) to be Process Mined: Bit2win, MuleSoft

Key Contact: Emanuele Mammoliti (Head of ICT)

2. Pulsee Customer Onboarding & Switching (Fornitura) — Digital-Native Retail

Pulsee is Axpo Italia's digital-only residential retail brand, replatformed on Heroku + Data Cloud + MuleSoft for scale. Every new-customer sign-up triggers a regulated fornitura-switching flow across the distributor (E-Distribuzione / SNAM), SAP billing, and Marketing Cloud — a single stuck switch drops the customer into a black hole and shows up as a Service Cloud case. AFPI mines the end-to-end onboarding event log to quantify switch-completion rate, drop-off variants, and case-generating patterns, feeding the Agentforce 1 upgrade with the real conformance map so the autonomous agent handles the 80% path and escalates the 20%. Metric moved: switch completion rate, days-to-first-invoice, abandoned-onboarding rate.

Key System(s) to be Process Mined: Heroku, MuleSoft

Key Contact: Salvatore Pinto (CEO Business Unit Consumer / Pulsee)

3. Service Case-to-Resolution — De-Risking the Agentforce 1 Upgrade

The $234k Agentforce 1 upgrade opp lands Autonomous Agents on top of an Einstein 1 Service Cloud that already handles both Axpo B2B (energy portfolios, PPA contract changes) and Pulsee B2C (billing disputes, switching support). Deploying Agentforce without knowing which case types are actually deflectable is a coin-flip. AFPI mines the case event log to expose case-type variant patterns, agent-touch counts, and self-service leakage — producing the "what should the agent take" whitelist that turns the upgrade from a feature buy into a measured deflection program. Metric moved: agent-deflectable case %, first-contact resolution, average handle time.

Key System(s) to be Process Mined: Service Cloud, MuleSoft

4. Field Service for Smartbox — Meter/Device Install and Maintenance

The "Axpo - Field Service for Smartbox" opp ($90k, stage 02) signals a field-service rollout for Axpo's Smartbox smart-metering / energy-management devices installed at customer premises. Every install, replacement, and firmware-related service call is an event chain across MuleSoft, Service Cloud, and dispatch — first-time-fix rates on regulated metering directly drive customer satisfaction and billing accuracy. AFPI mines the dispatch-to-close event log to expose reschedule loops, missing-part patterns, and technician-skill mismatches BEFORE Field Service goes live at scale. Metric moved: first-time-fix rate, mean time to install, truck-roll cost per install.
